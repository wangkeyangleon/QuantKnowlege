## PTOS-T 
寻找已经形成上升趋势，并且正在突破平台或者突破后缩量回踩的股票

### 1、基础股票池
每日收盘后扫描全市场，先排除：
- ST、退市整理；
- 停牌；
- 上市时间过短；
- 成交额过低；
- 价格过高导致2万元账户买不起100股；
- 一字涨跌停等无法成交股票。
可以设置为：
	上市交易日数 ≥ 250日
	20日平均成交额 ≥ 5000万元
	股价 ≤ 账户单票最大可买金额 ÷ 100

### 2、识别中长期上升趋势
判断条件：
- Close > MA20
- MA20 > MA60
- MA60斜率 > 0
- Close > MA120
- 过去60日相对沪深300收益 > 0

伪代码
trend_ok = (
    close > ma20
    and ma20 > ma60
    and ma60 > ma60_10_days_ago
    and close > ma120
)

#### 趋势候选池
符合条件后进入TREND_CANDIDATE
每天记录：
- 股票代码
- MA20、MA60、MA120
- 均线斜率
- 20日、60日涨幅
- 相对强度
- 所属板块

### 3、识别平台
识别三类平台

| 平台类型 |      周期 | 适用情况      |
| ---- | ------: | --------- |
| 短平台  |  20-40日 | 强趋势中的短期整理 |
| 中平台  | 40-120日 | 最适合趋势突破   |
| 长平台  |  6-12个月 | 大级别底部突破   |
首板重点做60-120日中期平台

**平台条件：**
- 平台宽度 ≤ 25%
- 平台内至少停留40个交易日
- 最高价附近至少触碰2次
- ATR逐步下降
- 近20日成交量低于前60日均量
- MA60走平或开始向上
平台顶部建议定义为：过去平台区间高点的95%分位数

满足以上条件后进入BASE_CONFIRMED

### 4、发现平台突破
系统每天检查平台股票是否发生突破
突破条件
- 收盘价 > 平台顶部 × 1.005
- 成交量 > 20日均量 × 1.5
- 收盘位于当天振幅上部
- 板块趋势不弱
- 突破距离不超过1.5倍ATR

伪代码：
breakout = (
    close > platform_top * 1.005
    and volume_ratio >= 1.5
    and close_position >= 0.65
    and distance_from_top <= 1.5 * atr
)

close_position = (收盘价 - 最低价) / (最高价 - 最低价)

突破后进入BREAKOUT_EVENT

### 5、等待回踩
从突破后的第一个交易日开始，最长检测10-15个交易日
每天检查三个锚点：
- 平台顶部
- MA10
- MA20
回踩条件：
- 最低价接近锚点，距离不超过2%或1倍ATR
- 收盘价没有有效跌破平台
- 回踩成交量低于突破日成交量
- 近3日平均成交量逐渐下降
- 没有连续放量长阴
伪代码
pullback_ok = (
    distance_to_anchor <= min(0.02, atr_pct)
    and close >= platform_top * 0.98
    and avg_volume_3d <= breakout_volume * 0.65
)

满足后进入 PULLBACK_READY

### 6、识别二次启动
设置两个触发器，满足一个即可

触发器A：低吸确认
- 当日最低价触碰平台或MA20
- 收盘重新站上MA5
- 收盘价高于前一日最高价
- 成交量高于近3日平均量

触发器B：小平台突破
- 回踩后横盘2—5日
- 收盘突破回踩期间局部高点
- 量比 ≥ 1.3
- 距离平台顶部不超过2倍ATR

满足后生成 PTOS_T_BUY_SIGNAL

### 完整发现路径
全市场扫描
→ 中期趋势成立
→ 形成60—120日平台
→ 放量突破平台
→ 3—15日内缩量回踩
→ 平台/MA20附近企稳
→ 再次放量启动
→ 买入信号

### 所需数据

| 数据        | 是否需要 |
| --------- | ---- |
| 日线行情      | 必须   |
| MA、ATR、平台 | 必须   |
| 财务数据      | 软评分  |
| 板块日线趋势    | 必须   |
| 分钟行情      | 可选   |
| 板块涨停扩散度   | 软评分  |
| 市场情绪周期    | 过滤仓位 |
| 新闻和题材催化   | 软评分  |

### 每天如何运行
- 15:10 更新日线行情
- 15:20 计算均线、ATR、平台和板块强度
- 15:30 扫描趋势候选
- 15:40 识别突破事件
- 15:45 更新回踩状态
- 15:50 生成次日观察清单
- 次日14:45 检查是否形成二次启动

PTOS-T可以在14:45附近做交易判断，但所有指标必须使用当时已知数据。

### 信号漏斗
全市场数量
→ 流动性合格
→ 趋势成立
→ 平台成立
→ 突破事件
→ 有效回踩
→ 二次启动
→ 可成交买入信号

每天记录，如果某一步突然变成0，就知道是哪条规则有问题


## 池子含义
|池子|含义|是否可以买|
|---|---|---|
|`TREND_PREPARATION`|已满足流动性、均线趋势、相对强度等预备条件，但没有形成合格平台|否|
|`BASE_CONFIRMED`|已形成合格的60/90/120日整理平台，等待突破|否|
|`BREAKOUT_EVENT`|当天刚发生有效突破，尚在等待未来1—3个完整交易日确认|否|
|`BREAKOUT_CONFIRMED`|突破已经确认，记录了独立的 `breakout_confirmed_date`|否|
|`PULLBACK_MONITORING`|从突破确认日之后开始监测缩量回踩和支撑锚点|否|
|`PULLBACK_READY`|已完成有效缩量回踩，等待二次启动触发器|否|
|`BUY_READY`|二次启动成立、止损和盈亏比校验通过，生成正式买入信号|可以进入下一交易日成交和组合风控|
